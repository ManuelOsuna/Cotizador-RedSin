Para portar tu aplicación REDSIN a iOS utilizando la Opción 2: Compose Multiplatform (Kotlin Multiplatform), puedes reutilizar más del 90% de todo el código de Jetpack Compose, base de datos Room, ViewModel y lógica de Gemini que acabamos de construir.

Aquí tienes la guía paso a paso y la hoja de ruta clara para crear y ejecutar tu versión de iOS:

Paso 1: Configurar tu Entorno de Desarrollo (Requisito para iOS)

Para compilar y empaquetar aplicaciones para iPhone (iOS), Apple requiere el uso de su sistema operativo:

  1. Consigue una Mac (MacBook, Mac Mini, iMac) con procesador Apple Silicon (M1/M2/M3) o Intel.

  2. Instala Xcode desde la Mac App Store (es el compilador oficial de Apple).

  3. Instala Android Studio en tu Mac para tener un entorno unificado donde puedas editar tanto la versión de Android como la de iOS juntas en Kotlin.

Paso 2: Crear la estructura compartida (Multiplatform)

La arquitectura de Compose Multiplatform divide el proyecto en tres carpetas principales:

  • composeApp/commonMain: Contiene tus pantallas (MainActivity.kt, UiComponents.kt), tus modelos de datos (QuoteEntities.kt), tus ViewModels y la lógica del servicio de Gemini. No tienes que reescribirlo, ¡se copia tal cual!

  • composeApp/androidMain: Contiene inicializadores específicos de Android.

  • composeApp/iosMain: Contiene el punto de entrada para iOS para renderizar la pantalla compartida en un contenedor de UIKit.

Puedes generar una plantilla limpia y oficial en un minuto usando el Kotlin Multiplatform Wizard seleccionando:

  • Plataformas: Android e iOS

  • Framework UI: Share UI (Compose Multiplatform)

Paso 3: Migrar tu código de REDSIN a la carpeta compartida (commonMain)

Copia tus archivos del proyecto de Android a la carpeta común para que corran en ambos sistemas:

  1. Vistas y UI: Copia todo el contenido de MainActivity.kt (mantenlo como funciones de Compose ordinarias listos para renderizar) y UiComponents.kt.

  2. Base de Datos (Room): Room es ahora compatible con Kotlin Multiplatform. Tus entidades (Quote, QuoteItem, CatalogProduct) y tu base de datos se quedan igual; solo ajustas el inicializador para proveer la base de datos de manera nativa en cada plataforma (SQLite en Android y SQLite nativo en iOS).

  3. Lógica del Servicio de Gemini: Copia GeminiService.kt. Al usar bibliotecas multiplataforma como Ktor o la SDK oficial de Gemini para KMP, funcionará idéntico en tu iPhone.

Paso 4: Adaptar las funciones específicas de iOS

En nuestro código de Android, hay APIs específicas del sistema de Google (como el gestor de impresión PrintManager y compartir portapapeles). En Kotlin Multiplatform, estas funciones se resuelven usando la estructura expect / actual.

Por ejemplo, para que el botón de Imprimir / Guardar PDF funcione de forma nativa en tu iPhone, la implementación en Kotlin dentro de la carpeta iosMain llamará a las herramientas nativas de iOS de esta manera:

code Kotlin
// En iosMain (Kotlin) llamamos al controlador nativo de impresión de Apple:
import platform.UIKit.UIPrintInteractionController
import platform.UIKit.UIPrintInfo
import platform.Foundation.NSURL
import platform.WebKit.WKWebView

actual fun printHtml(htmlString: String, jobName: String) {
    val printController = UIPrintInteractionController.sharedPrintController()
    val printInfo = UIPrintInfo.printInfoAndReturnType()
    printInfo.outputType = platform.UIKit.UIPrintInfoOutputGeneral
    printInfo.jobName = jobName
    printController.printInfo = printInfo
    
    val webView = WKWebView()
    webView.loadHTMLString(htmlString, null)
    
    // Asignamos el formateador de impresión de la webview nativa de iOS
    printController.printFormatter = webView.viewPrintFormatter()
    printController.presentAnimated(true, completionHandler = null)
}

Paso 5: Ejecutar la aplicación en tu iPhone

  1. Abre tu nuevo proyecto multiplataforma en Android Studio.

  2. Conecta tu iPhone a tu Mac por USB (o abre el Simulador de iPhone en Xcode).

  3. En Android Studio, selecciona la configuración de ejecución iosApp en la barra superior.

  4. Presiona el botón verde de Play (Run).

  5. ¡Listo! Android Studio compilará el código de Kotlin en código nativo de Apple y la aplicación de REDSIN se abrirá en tu iPhone con el nuevo diseño, logo y todas las opciones de PDF activadas.

Para tener tu Cotizador de REDSIN funcionando al 100% en tu iPhone, iPad, laptop o cualquier dispositivo sin necesidad de instalar nada desde la App Store, la mejor opción es una Web App (Aplicación Web Responsiva).

He preparado para ti el código completo de una PWA (Progressive Web App) en un solo archivo. Esto significa que puedes subirla gratis a internet, o simplemente guardar el archivo en tu teléfono y abrirlo. En tu iPhone aparecerá una pantalla idéntica a la que diseñamos para Android, optimizada para iOS y con la función de Imprimir / Guardar PDF nativa de Apple.

Aquí tienes los pasos sencillos para activarla y el código que necesitas:

  2. Crea un archivo nuevo y nómbralo index.html.
